docker 在ubuntu 14.04(Trusty)

1 sudo apt-key adv --keyserver hkp://p80.pool.sks-keyservers.net:80 --recv-keys 58118E89F3A912897C070ADBF76221572C52609D 2 创建文件 sudo touch /etc/apt/sources.list.d/docker.list 将 deb https://apt.dockerproject.org/repo ubuntu-trusty main 添加到 docker.lits 3 sudo apt-get update 4 Purge the old repo if it exists. $ apt-get purge lxc-docker 5 Verify that apt is pulling from the right repository. $ apt-cache policy docker-engine 6 From now on when you run apt-get upgrade, apt pulls from the new repository.

7 sudo apt-get install docker-engine

8 Start the docker daemon #以守护进程启动 sudo service docker start/stop

ps: apt-get dist-upgrade #升级软件 自动管理依赖

Optional configurations

1 create a Docker group docker daemon 绑定了一个Unix socket而不是Tcp port。默认状态下Unix socket的所有者是root用户。 其它用户使用sudo能够访问它。为了避免在使用docker命令时每次都加sudo,创建一个叫docker的group，并将 用户加入改组。。重启后生效

$ sudo usermod -aG docker your_username

By default that Unix socket is owned by the user root and other users can access it with sudo. For this reason, docker daemon always runs as the root user.

To avoid having to use sudo when you use the docker command, create a Unix group called docker and add users to it. When the docker daemon starts, it makes the ownership of the Unix socket read/writable by the docker group.
